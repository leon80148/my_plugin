# 快速上手說明文件產生器

產出讓新手 **5-10 分鐘內成功** 的最小可行說明文件。

## 三大目標（必達）

1. **第一次就成功**：照做能完成第一個可驗證成果
2. **下一步清楚**：完成後知道接下來做什麼
3. **卡住可自救**：遇錯能用 Troubleshooting 快速排除

## 章節骨架（固定結構）

```markdown
# [產品/功能名稱]

## 這是什麼
<!-- 1-2 句說明這個功能能幫使用者做什麼 -->

## 快速開始
<!-- 最短路徑完成第一個成果，必須有成功判定 -->

## 常見任務
<!-- 2-3 個最常用的 How-to -->

## 卡住了怎麼辦
<!-- 症狀導向的 Troubleshooting -->
```

## 步驟寫法（硬規則）

### 必須遵守

- **一句一動作**：每步只做一件事
- **動作 + 回饋**：告訴讀者「做什麼」+「會看到什麼」
- 用 **粗體** 標示按鈕名稱、選單路徑、重要欄位

### 正確範例

```markdown
1. 點選 **設定 → 帳號**。
2. 在 **API Key** 欄位貼上金鑰。
3. 按 **儲存**，右上角會出現「已更新」提示。
```

### 禁止

- 多個動作塞一句：「打開設定然後找到帳號把金鑰貼進去」
- 模糊詞：「處理一下」「設定好就可以」「按照指示操作」

## 提示框規範

固定使用以下四種：

```markdown
> ✅ **最佳做法**：建議的正確做法

> ⚠️ **注意**：不可逆或高風險操作警告

> 🛠️ **小技巧**：省力捷徑

> 🧯 **卡住了？** 立刻可嘗試的排除方法
```

## 圖片決策規則

依以下條件判斷是否需要圖片：

| 條件 | 需要圖片 | 圖片類型 |
|------|---------|---------|
| UI 導航超過 2 層 | ✅ 必須 | 定位圖 |
| 成功判定不直觀（需看狀態/log） | ✅ 必須 | 成功畫面圖 |
| 高風險/不可逆操作 | ✅ 必須 | 確認對話框圖 |
| 實體操作（接線、按鍵、安裝） | ✅ 必須 | 步驟圖 |
| 常見錯誤點 | 🔶 建議 | 正確 vs 錯誤對照圖 |
| 按鈕位置明顯且單一 | ❌ 不需要 | — |

詳細圖片規格見 `references/image-rules.md`。

## 圖片需求輸出格式

每個需要圖片的位置，輸出以下區塊：

```markdown
<!-- IMAGE_NEEDED
type: [where | success | compare | step]
description: [這張圖要拍/畫什麼]
key_elements: [必須出現的元素，用逗號分隔]
annotation: [需要的標註：框選/箭頭/編號]
alt_text: [替代文字]
caption: [圖說，包含「看哪裡」+「成功判定」]
-->
```

### 範例

```markdown
<!-- IMAGE_NEEDED
type: where
description: 設定頁面的 API Key 輸入區塊
key_elements: 設定選單, 帳號分頁, API Key 欄位, 儲存按鈕
annotation: 框選 API Key 欄位，箭頭指向儲存按鈕
alt_text: 設定頁的帳號區塊，顯示 API Key 輸入欄位與儲存按鈕位置
caption: 在「設定 → 帳號」頁面找到 API Key 欄位，貼上後按「儲存」
-->
```

## 輸出品質檢核

完成文件後自我檢查：

- [ ] 「快速開始」能在 5-10 分鐘內完成
- [ ] 每個步驟都有成功判定
- [ ] UI 導航 > 2 層的地方都有圖片需求
- [ ] 高風險操作有 ⚠️ 警告
- [ ] Troubleshooting 是「症狀 → 解法」格式
