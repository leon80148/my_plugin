# UI/UX 設計與使用者體驗優化

## 優化新手體驗 (Onboarding)

### 用戶旅程審視
```
請扮演一位第一次使用本應用的用戶，
描述從打開 App、註冊帳號到完成第一個任務的過程，
有哪些步驟可能讓人感到困擾或中途放棄？
```

### 常見 Onboarding 問題
- 註冊流程過於冗長
- 權限請求彈窗突兀
- 引導提示不清楚
- 價值主張傳達不足
- 過早要求付費

### 優化建議
- 精簡表單字段（只要必要資訊）
- 加入進度指示器
- 提供跳過選項
- 延後非必要權限請求
- 讓用戶先體驗核心價值再要求註冊

## 建立反饋迴路

### 收集用戶反饋
- 懸浮的「意見反饋」按鈕
- 完成關鍵操作後的簡短調查
- NPS (Net Promoter Score) 評分

### 給用戶正向反饋
- 完成任務時的打勾動畫
- 鼓勵性文案：「幹得好！你又向目標邁進了一步」
- 進度視覺化

## 遊戲化設計

### 常見遊戲化元素
- **積分系統**：完成任務獲得點數
- **徽章成就**：達成里程碑解鎖
- **排行榜**：與好友或全球用戶競爭
- **連續簽到**：每日使用獎勵
- **等級系統**：長期投入的成就感

### Hook 模型（習慣養成四階段）

1. **觸發 (Trigger)**
   - 外部觸發：推播通知、Email、廣告
   - 內部觸發：情緒、習慣、需求

2. **行動 (Action)**
   - 降低使用門檻
   - 簡化操作步驟
   - 清楚的行動呼籲 (CTA)

3. **可變獎勵 (Variable Reward)**
   - 社交獎勵：他人的認可、讚、評論
   - 狩獵獎勵：隨機驚喜、新內容發現
   - 自我獎勵：完成感、技能提升

4. **投入 (Investment)**
   - 建立個人資料
   - 累積社交關係
   - 儲存內容/進度
   - 自定義設定

### Hook 模型分析提示
```
請用 Hook 模型分析我的產品：
{產品描述}

評估我們在四個環節（觸發、行動、可變獎勵、投入）的表現，
並給出具體的改進建議。
```

## 跨平台一致性

### iOS vs Android 差異
| 項目 | iOS | Android |
|------|-----|---------|
| 返回操作 | 左滑返回 | 實體/虛擬返回鍵 |
| 導航位置 | 底部 Tab Bar | 頂部 Tab 或漢堡選單 |
| 動畫風格 | 滑動過渡 | Material Design 波紋 |
| 按鈕位置 | 導航列右側 | 浮動操作按鈕 (FAB) |

## 微文案 (Microcopy)

### 優化原則
- 使用用戶語言，避免技術術語
- 保持整體語調一致
- 錯誤訊息要友善且有幫助
- 按鈕文字要動作導向

### 提示範本
```
請幫我優化以下錯誤訊息，讓它更友善且有幫助：
原始訊息：「Error 500: Internal Server Error」
```
