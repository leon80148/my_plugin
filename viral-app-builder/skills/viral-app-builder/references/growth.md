# 成長黑客策略與用戶成長

## 什麼是成長黑客

聚焦於快速試驗和非常規策略來達成用戶數指數級成長的方法論。
核心理念：小步快跑、資料驅動、持續實驗。

## 打造病毒循環 (Viral Loop)

### 病毒傳播機制設計
```
如何讓現有用戶在使用產品的過程中自然地推廣產品？
請針對 {產品類型} 設計病毒傳播機制。
```

### 雙邊推薦獎勵計畫
```
請設計用戶推薦獎勵計畫：
1. 推薦人獎勵內容
2. 被推薦人獎勵內容
3. 邀請碼/連結系統
4. 獎勵發放條件
5. 防濫用措施
```

### UGC 激勵活動
```
有什麼活動能讓用戶自發創作並分享內容，
從而吸引新用戶？

產品類型：{產品描述}
目標用戶：{用戶畫像}
```

**活動類型建議：**
- 話題標籤挑戰（#30天打卡挑戰）
- 創作比賽（最佳作品獎勵）
- 分享有禮（分享截圖獲得獎勵）
- 里程碑慶祝（第1000位用戶特別獎）

## AARRR 漏斗優化

### Acquisition（獲客）
- 優化 App Store 頁面（ASO）
- 社群媒體內容行銷
- 付費廣告投放
- 口碑推薦

### Activation（激活）
- 簡化註冊流程
- 提供社媒一鍵登入
- 讓用戶先體驗再註冊
- 清楚傳達核心價值

### Retention（留存）
- 推播通知（提供價值，非騷擾）
- 週/月度挑戰目標
- 社交元素（排行榜、社群）
- 個人化內容推薦

### Referral（推薦）
- 雙邊獎勵機制
- 簡化分享流程
- 社交分享按鈕
- 邀請碼追蹤

### Revenue（變現）
- 見 [monetization.md](monetization.md)

### 漏斗分析請求
```
我的產品各階段數據如下：
- 下載：{數字}
- 註冊完成：{數字}
- 首次使用核心功能：{數字}
- 7日留存：{數字}
- 付費轉化：{數字}

哪個環節流失率最高？有什麼改進建議？
```

## A/B 測試與實驗

### 實驗設計請求
```
我們想比較 {方案A} 和 {方案B}，
請問該如何設定實驗組和控制組，
以及衡量成功的指標？
```

### 實驗設計要點
1. **明確假設**：「我相信 {改變} 會導致 {結果}」
2. **單一變量**：一次只測試一個變化
3. **足夠樣本**：確保統計顯著性
4. **時間長度**：考慮週期性因素
5. **成功指標**：事先定義評判標準

### 結果分析請求
```
A/B 測試結果：
- 版本 A（控制組）：{數據}
- 版本 B（實驗組）：{數據}

請分析：
1. 哪組表現更好？
2. 差異是否顯著？
3. 可能的原因？
4. 後續行動建議？
```

## 用戶洞察與社群經營

### 行銷文案構思
```
我們的產品想訴求 {目標用戶}，
有什麼行銷語或故事角度能引起他們共鳴？
```

### 內容策略規劃
```
針對 {產品類型}，
請建議社群內容策略：
1. 適合的貼文類型
2. 發布頻率
3. 互動活動建議
```

### 種子用戶經營
- 建立 VIP 用戶群組
- 舉辦線上/線下交流活動
- 邀請參與產品測試
- 給予專屬獎勵或功能
