# 隱私合規指南

本文件提供 App 開發中常見的隱私法規要求與實作模式。

---

## 法規總覽

| 法規 | 適用地區 | 適用對象 | 重點要求 |
|------|---------|---------|---------|
| GDPR | 歐盟/EEA | 處理歐盟居民資料的任何組織 | 合法基礎、資料最小化、DPIA |
| 台灣個資法 (PDPA) | 台灣 | 蒐集台灣居民個資的組織 | 告知義務、特定目的、當事人權利 |
| CCPA/CPRA | 美國加州 | 符合門檻的企業 | 知情權、刪除權、退出銷售 |
| LGPD | 巴西 | 處理巴西居民資料的組織 | 與 GDPR 類似 |
| PIPL | 中國 | 處理中國居民個資的組織 | 本地化儲存、同意 |

---

## GDPR 合規檢查清單

### 資料處理原則

- 合法基礎：確認每項資料處理有合法基礎（同意/合約/合法利益/法律義務）
- 目的限制：明確定義資料使用目的，不做目的外使用
- 資料最小化：只蒐集必要的資料
- 正確性：提供機制讓用戶更正資料
- 儲存限制：設定資料保留期限，過期自動刪除
- 完整性與機密性：加密傳輸與儲存

### 使用者權利

| 權利 | 說明 | 實作方式 |
|------|------|---------|
| 存取權 | 用戶可取得其資料副本 | 資料匯出功能（JSON/CSV） |
| 更正權 | 用戶可修正錯誤資料 | 個人資料編輯頁面 |
| 刪除權 | 用戶可要求刪除資料 | 帳號刪除功能 + 資料清除 |
| 可攜權 | 用戶可以機器可讀格式取得資料 | API 匯出或一鍵下載 |
| 反對權 | 用戶可反對特定處理 | 偏好設定中的退出選項 |
| 限制處理 | 用戶可暫停資料處理 | 帳號凍結功能 |

### DPIA 觸發條件

以下情境必須執行 DPIA：
- 大規模處理敏感資料（健康、生物識別）
- 自動化決策/剖析影響用戶
- 大規模公共區域監控
- 使用新技術（AI/ML）處理個人資料

## 台灣個資法 (PDPA)

### 核心要求

| 要求 | 說明 | 實作 |
|------|------|------|
| 告知義務 | 蒐集前須告知目的、類別、使用期間等 | 隱私政策 + 首次使用同意畫面 |
| 特定目的 | 必須有明確的蒐集目的 | 每項資料欄位對應目的碼 |
| 同意 | 敏感資料需書面同意 | 明確勾選 + 簽名（敏感資料） |
| 安全維護 | 適當的安全措施 | 加密、存取控制、稽核日誌 |
| 當事人權利 | 查閱、更正、刪除、停止蒐集/處理/利用 | 客服管道 + 自助功能 |

### 個資法特定目的代碼（常見）

| 代碼 | 目的 | 適用場景 |
|------|------|---------|
| 040 | 行銷 | 推播通知、EDM |
| 069 | 契約/類似契約 | 帳號註冊、訂購 |
| 090 | 消費者保護 | 客訴處理 |
| 136 | 資通安全 | 登入紀錄、異常偵測 |
| 157 | 調查統計 | 匿名數據分析 |
| 181 | 其他經營合理用途 | 產品改善 |

## App Store 隱私要求

### Apple App Store

| 要求 | 說明 |
|------|------|
| 隱私營養標籤 | 在 App Store Connect 填寫資料蒐集類型 |
| App Tracking Transparency (ATT) | 追蹤用戶需先取得授權 |
| 隱私政策連結 | App 內與商店頁面都必須有 |
| 資料最小化 | 審核時會檢查是否蒐集過多資料 |
| 帳號刪除 | 提供帳號建立的 App 必須提供刪除功能 |

### Google Play Store

| 要求 | 說明 |
|------|------|
| 資料安全性專區 | 填寫資料蒐集與分享聲明 |
| 隱私政策 | 必須提供且從 App 內可存取 |
| 敏感權限 | 位置、相機等需說明用途 |
| 兒童保護 | 面向兒童的 App 有額外限制 |
| 帳號刪除 | 2024 起必須提供帳號與資料刪除 |

## CCPA/CPRA（加州）

### 適用門檻（滿足任一）

- 年營收 > 5M
- 處理 > 100,000 加州居民資料
- 50%+ 營收來自販售/分享個人資料

### 核心權利

| 權利 | 實作 |
|------|------|
| 知情權 | 揭露蒐集了哪些資料 |
| 刪除權 | 提供刪除管道 |
| 退出銷售/分享 | Do Not Sell My Info 連結 |
| 不歧視 | 行使權利不影響服務品質 |
| 更正權 (CPRA) | 允許更正錯誤資料 |
| 限制敏感資料 (CPRA) | 限制敏感資料使用 |

## 實作模式

### CMP（同意管理平台）整合

同意管理的核心流程：
1. 偵測用戶地區
2. 依地區法規顯示同意選項（GDPR 預設 opt-out）
3. 記錄用戶偏好
4. 根據偏好啟用/停用 SDK（Analytics、Marketing 等）
5. 提供設定頁讓用戶隨時更改偏好

### 資料最小化模式

| 原則 | 實作 |
|------|------|
| 只蒐集需要的 | 註冊只要 email，其他延後蒐集 |
| 匿名化統計 | 分析用匿名 ID，不綁個人 |
| 自動過期 | 非活躍帳號 N 個月後刪除 |
| 邊緣處理 | 可以在裝置上處理的不送到伺服器 |

### 匿名化技術

| 技術 | 適用場景 | 強度 |
|------|---------|------|
| 假名化 | 內部分析（可重新識別） | 中 |
| 雜湊 | Email 比對（不可逆） | 高 |
| K-匿名 | 公開資料集 | 高 |
| 差分隱私 | 統計查詢 | 最高 |
| 資料遮罩 | 顯示部分資料（手機 ***-1234） | 低 |
