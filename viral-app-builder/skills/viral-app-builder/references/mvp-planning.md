# MVP 功能規劃與原型設計

## 聚焦核心功能

### MoSCoW 優先級框架
- **Must-have**：沒有就無法運作的核心功能
- **Should-have**：重要但可稍後實現
- **Could-have**：錦上添花的功能
- **Won't-have**：這個版本不做

### ICE/RICE 評分法
對每個功能評估：
- **Impact**：對用戶/業務的影響力 (1-10)
- **Confidence**：實現後能達成預期的信心 (1-10)
- **Ease**：實現的容易程度 (1-10)
- **Reach**：影響的用戶數量（RICE 額外項目）

### 提示範本

**基礎 MVP 規劃：**
```
我想開發一款 {產品描述} 的 App，
請問 MVP 應包含哪些核心功能？
請列出：
1. 必要功能（無法刪除）
2. 重要功能（可稍後加）
3. 次要功能（暫不實現）
```

**框架套用：**
```
請用 MoSCoW 方法對以下功能分類：
{功能列表}
並說明分類理由。
```

**資源限制聚焦：**
```
假設只有7天開發時間和500美元預算，
{產品描述} 應該優先實現哪些功能？
```

## 概念驗證方法

在不開發完整產品的情況下驗證需求：

1. **Landing Page 測試**：建立宣傳頁面收集 Email
2. **問卷調查**：驗證痛點和付費意願
3. **預先登記**：測量真實需求強度
4. **Wizard of Oz**：手動模擬後端服務
5. **Concierge MVP**：人工提供服務驗證流程

```
有什麼快速的方法在不開發完整產品的情況下
驗證用戶對「{產品概念}」的興趣？
```

## 原型設計

### 線框圖設計提示
```
我在開發一款 {產品類型} 應用。
請幫我設計主畫面的線框：
- {描述你需要的元素}
- {描述互動方式}
- {描述資訊層級}
```

### 用戶流程設計
```
請設計從「下載 App」到「完成第一個核心任務」的用戶流程。
每個步驟列出：畫面名稱、主要操作、可能的阻礙點。
```

## MVP 檢核清單

- [ ] 功能清單精簡到無法再刪減
- [ ] 每個功能都對應一個明確的用戶需求
- [ ] 有明確的成功指標 (KPI)
- [ ] 有驗證假設的計畫
- [ ] 預留迭代改進的空間
