# 程式開發與 Claude 協同寫碼

## 撰寫高品質提示語

### 有效的開發請求結構
1. **技術棧**：明確指定語言、框架、工具
2. **功能需求**：具體列出要實現的功能點
3. **約束條件**：效能、安全、最佳實踐要求
4. **輸出格式**：代碼結構、註解要求

### 提示範本
```
我需要用 {框架} 開發 {功能描述}。

需求：
1. {功能點1}
2. {功能點2}
3. {功能點3}

請確保：
- 完整的錯誤處理和邊界情況
- 關注效能表現
- 遵循 {框架} 的最佳實踐
- 包含詳細註解

技術約束：
- 使用 {狀態管理方案}
- 遵循 {架構模式}
```

### 避免的提示方式
❌ 「幫我寫一個聊天應用的代碼」（太籠統）
✅ 「用 Flutter + Riverpod 開發聊天介面，包含訊息列表、輸入框、發送按鈕，支援即時更新」

## 審查 AI 代碼

### 審查要點
- [ ] 理解每一行代碼的用途
- [ ] 檢查邏輯是否符合需求
- [ ] 確認錯誤處理完整
- [ ] 確認邊界情況有考慮
- [ ] 確認符合框架最佳實踐
- [ ] 確認沒有安全漏洞

### 代碼解釋請求
```
請逐步解釋以下程式碼的作用，
尤其是第 {行號} 在做什麼，有沒有可能的問題？

{代碼片段}
```

### 代碼審查請求
```
請審查以下代碼，指出：
1. 任何不符合最佳實踐的地方
2. 可能的 bug 或邏輯漏洞
3. 性能優化建議
4. 安全性問題

{代碼片段}
```

### 性能優化請求
```
以下函式在大量數據下執行緩慢，
有沒有方法可以改進效率？

{代碼片段}
```

## 跨平台開發

### 技術棧指定範例

**React Native (Expo)：**
```
請產生 React Native (Expo) 的 TypeScript 代碼，
使用 React Navigation 進行頁面路由，
使用 React Query 處理 API 請求。
```

**Flutter：**
```
請產生 Flutter 代碼，
使用 Riverpod 進行狀態管理，
遵循 MVVM 架構模式。
```

## 開發流程管理

### 循序漸進原則
1. 拆解任務為小模組
2. 每次請求一個功能點
3. 驗證後再進行下一步
4. 使用 Git 追蹤變更

### 上下文提示
```
我們現在修改的是 {檔案名稱}，
這個檔案負責 {功能描述}。
接下來要實現 {具體任務}。
```

## 即時除錯

### 除錯請求範本
```
在 {操作描述} 後應用 {錯誤現象}，
錯誤日誌顯示：{錯誤訊息}
以下是相關程式碼：

{代碼片段}

可能原因是什麼？如何修復？
```

### 系統性問題分析
```
我的應用有以下問題：{問題描述}
這是整體架構：{架構說明}
請分析可能的根本原因，並建議解決方向。
```
