# 多 Agent 協作內容審查團隊

## 概述

本插件提供一個由 6 位 AI Agent 組成的虛擬內容審查團隊，透過結構化的 4 輪討論流程，確保專業內容的品質、準確性、可讀性與傳播力。

## 團隊角色

| 角色 | Agent 檔案 | 職責 | 審查焦點 |
|------|-----------|------|----------|
| 🎯 主持人 | `agents/facilitator_agent.md` | 引導討論、協調衝突、裁決分歧 | 流程把關、整體品質 |
| 📝 醫學記者 | `agents/medical_journalist_agent.md` | 撰寫初稿、執行修改 | 內容結構、敘事品質 |
| 🔬 科學守門人 | `agents/science_gatekeeper_agent.md` | 事實查核、證據審查 | 專業準確性、引用品質 |
| 💚 病患代言人 | `agents/patient_advocate_agent.md` | 讀者視角審查 | 可讀性、共感度、實用性 |
| 😈 魔鬼代言人 | `agents/devil_advocate_agent.md` | 挑戰假設、找出漏洞 | 邏輯漏洞、偏見、風險 |
| 📈 流量獵人 | `agents/traffic_hunter_agent.md` | 傳播優化、SEO | 標題吸引力、分享率 |

## 支援領域

| 領域 | 說明 | 角色名稱調整 |
|------|------|-------------|
| **醫療**（預設） | 衛教文章、疾病科普、藥物知識 | 原始角色名稱 |
| **科技** | 技術文章、產品評測、趨勢分析 | 科學守門人 → 技術審核員 |
| **財經** | 投資觀念、理財知識、經濟分析 | 科學守門人 → 合規審核員 |
| **法律** | 法律常識、權益解說、判例分析 | 科學守門人 → 法律審核員 |
| **行銷** | 品牌文案、行銷策略、市場分析 | 科學守門人 → 品牌一致性審核 |

詳細的領域預設配置：`references/domain-presets.md`
各領域的角色名稱對照：`references/role-mapping.md`

## 4 輪審查流程

```
第 0 輪：初稿撰寫
├── 醫學記者獨立撰寫初稿
├── 主持人準備審查議程
└── 輸出：draft_v0.md

第 1 輪：體檢（60 分鐘）
├── 全團隊審查初稿
├── 科學守門人：事實與證據
├── 病患代言人：可讀性與共感
├── 魔鬼代言人：邏輯與風險
├── 流量獵人：傳播與吸引力
├── 主持人：協調與裁決
└── 輸出：round1.md（會議記錄）

第 2 輪：改寫（45 分鐘）
├── 醫學記者根據回饋修改
├── 各角色確認修改是否到位
├── 主持人追蹤改進項目
└── 輸出：draft_v1.md + round2.md

第 3 輪：包裝（45 分鐘）
├── 流量獵人：標題/SEO 優化
├── 病患代言人：最終可讀性確認
├── 魔鬼代言人：最終風險檢查
└── 輸出：draft_v2.md + round3.md

第 4 輪：定稿（30 分鐘）
├── 全團隊最終審核
├── 檢查清單逐項確認
├── 主持人簽核
└── 輸出：final.md + round4.md
```

## 使用方式

### 啟動討論

使用 `/discuss` 指令啟動團隊討論：

```
/discuss [主題] --domain [領域] --depth [深度]
```

參數說明：

| 參數 | 說明 | 預設值 |
|------|------|--------|
| 主題 | 要討論的文章主題（必填） | — |
| `--domain` | 領域（medical/tech/financial/legal/marketing） | medical |
| `--depth` | 討論深度（quick=2輪 / standard=4輪 / deep=4輪+追加） | standard |

### 擷取圖表

使用 `/extract-figures` 指令從文章中提取圖表需求。

## 指令

| 指令 | 檔案 | 說明 |
|------|------|------|
| `/discuss` | `commands/discuss.md` | 啟動多角色團隊討論 |
| `/extract-figures` | `commands/extract-figures.md` | 從文章擷取圖表需求 |

## 輸出品質標準

每篇經審查的文章應達到：

| 面向 | 標準 |
|------|------|
| 專業準確性 | 科學守門人確認所有事實正確、引用可靠 |
| 可讀性 | 病患代言人確認一般讀者可理解 |
| 邏輯完整性 | 魔鬼代言人確認無邏輯漏洞 |
| 傳播潛力 | 流量獵人確認標題吸引、SEO 到位 |
| 安全性 | 免責聲明到位、無誤導性內容 |
| 散文品質 | 敘事流暢、結構清晰、語氣一致 |

## 快速討論模式

當使用 `--depth quick` 時，流程簡化為 2 輪：

```
第 0 輪：初稿撰寫（同 standard）
第 1 輪：快速體檢（30 分鐘）
├── 所有角色同時提供回饋
├── 主持人彙整重點
└── 醫學記者直接修改
輸出：final.md
```

適用場景：時效性高的內容（新聞稿、即時回應）、內部文件草稿。

## 深度討論模式

當使用 `--depth deep` 時，在 standard 4 輪之後追加：

```
第 5 輪：外部審查模擬（30 分鐘）
├── 模擬讀者留言反應
├── 模擬同業/競品觀點
├── 預測可能的爭議點
└── 輸出：external-review.md

第 6 輪：最終修訂
├── 根據外部審查調整
├── 最終定稿
└── 輸出：final-deep.md
```

適用場景：高影響力的公開文章、學術級科普內容、可能引發爭議的主題。

## 故障排除

| 問題 | 原因 | 解決 |
|------|------|------|
| Agent 回覆過長導致截斷 | 單次回覆超過 token 限制 | 拆分為多段回覆，或用 `--depth quick` |
| 角色觀點過於相似 | System prompt 區分度不夠 | 加強各 agent.md 中的角色差異指令 |
| 討論流程停滯 | 主持人未適時推進 | 在 facilitator_agent.md 中加入時間限制提示 |
| 領域術語不正確 | 非該領域的預設配置 | 使用 `--domain` 指定正確領域 |
