# 領域預設配置（Domain Presets）

本文件定義各內容領域的預設團隊配置。在 `/discuss` 指令中透過 `$DOMAIN` 參數選擇領域，系統將自動套用對應配置。

---

## 醫療衛教（Medical Education）— 預設

- **領域代碼**：`medical`
- **證據標準**：EBM 分級（A/B/C/D），優先引用系統性回顧與統合分析
- **必要角色強調**：科學守門人（醫學準確性最後防線）
- **免責聲明**：醫療免責（「本文僅供參考，不能取代專業醫療建議」）
- **目標讀者**：醫療同業（醫師/藥師/護理師）+ 關注健康的高知識份子
- **主要平台**：Facebook（長文）、Instagram（資訊圖表）、YouTube（解說影片）
- **內容基調**：專業 KOL 分析文風格，有臨床溫度
- **參考框架**：PICO 架構、GRADE 系統
- **證據來源**：PubMed、Cochrane Reviews、UpToDate、各醫學會指引
- **審查重點**：藥物名稱與劑量、統計數據出處、治療建議適用性、診斷標準準確性

## 科技教育（Tech Education）

- **領域代碼**：`tech`
- **證據標準**：官方文檔 + 基準測試 + 同儕審查技術論文
- **必要角色強調**：技術審查員（技術準確性與版本正確性）
- **免責聲明**：技術免責（「技術內容可能因版本更新而有所變動，請以官方文檔為準」）
- **目標讀者**：開發者 + 技術管理者 + 有興趣的非技術人員
- **主要平台**：Dev.to、Medium、Twitter/X、Hacker News、Reddit
- **內容基調**：技術深度兼具實用性，有工程師視角
- **參考框架**：技術文檔標準、RFC 規範、軟體工程最佳實踐
- **證據來源**：官方文檔、GitHub Repositories、RFC 文件、ACM/IEEE 論文、基準測試報告
- **審查重點**：程式碼正確性、版本相容性、安全性考量、效能宣稱的準確性

## 財經分析（Financial Analysis）

- **領域代碼**：`financial`
- **證據標準**：監管文件 + 審計報告 + 經同儕審查的經濟研究
- **必要角色強調**：合規審查員（法規合規與數據準確性）
- **免責聲明**：投資免責（「本文不構成投資建議，投資有風險，請審慎評估」）
- **目標讀者**：投資人 + 財務專業人員 + 關注理財的一般大眾
- **主要平台**：LinkedIn、Twitter/X、YouTube、Podcast
- **內容基調**：數據驅動的分析風格，客觀中立
- **參考框架**：財務分析框架、風險評估模型、監管合規標準
- **證據來源**：SEC/金管會公告、央行報告、審計財報、券商研究報告、經濟期刊
- **審查重點**：財務數據準確性、法規引用正確性、風險揭露充分性、利益衝突聲明

## 法律科普（Legal Education）

- **領域代碼**：`legal`
- **證據標準**：判例法 + 法條 + 法學評論
- **必要角色強調**：法律準確性審查（法條引用與適用性）
- **免責聲明**：法律免責（「本文僅供法律知識參考，不構成法律意見，具體案件請諮詢律師」）
- **目標讀者**：一般民眾 + 法律從業者 + 企業法務
- **主要平台**：LinkedIn、部落格、Podcast、電子報
- **內容基調**：白話法律、有溫度的法律解析
- **參考框架**：法條體系、判例分析架構、法律推理方法
- **證據來源**：司法院判決書、法條資料庫、律師公會指引、法學期刊、法律評論
- **審查重點**：法條引用正確性、判例適用性、法律效力說明、時效與管轄權提醒

## 行銷內容（Marketing Content）

- **領域代碼**：`marketing`
- **證據標準**：市場研究 + 用戶數據 + 產業報告
- **必要角色強調**：品牌一致性審查（品牌調性與訊息一致性）
- **免責聲明**：行銷免責（「本文所述效果因個人/企業情況而異」）
- **目標讀者**：目標客群 + 行銷從業者 + 品牌經營者
- **主要平台**：依品牌定位選擇（全平台策略）
- **內容基調**：品牌聲音一致、有說服力但不過度推銷
- **參考框架**：行銷漏斗、用戶旅程地圖、品牌定位策略
- **證據來源**：市場研究報告、用戶調研數據、Google Analytics、產業白皮書
- **審查重點**：品牌調性一致性、宣稱真實性、公平交易法合規、用戶隱私保護

## 如何新增領域

若需新增自訂領域，請按以下格式撰寫：

```markdown
## [領域名稱]（[English Name]）

- **領域代碼**：`[code]`
- **證據標準**：[該領域的權威證據類型]
- **必要角色強調**：[最關鍵的審查角色]
- **免責聲明**：[領域專屬免責聲明]
- **目標讀者**：[主要 + 次要讀者群]
- **主要平台**：[最適合的傳播平台]
- **內容基調**：[寫作風格描述]
- **參考框架**：[領域專用的分析框架]
- **證據來源**：[權威資料庫與來源]
- **審查重點**：[該領域的審查要項]
```

新增後，請同步更新 `role-mapping.md` 中的角色對照表。
