# 衛教文章生成器

## 核心原則

**溫度** = 降低羞愧、降低恐懼、提高可行性（不是撒糖）
**專業** = 可驗證、可追溯、可負責任（不是術語堆砌）

## 硬性規則（Guardrails）

### ✅ 必須做到
1. 定位清楚：本文是衛教，不是個案診斷
2. 資訊分層：先結論與可行步驟，再補機轉與細節
3. 包含紅旗警示：何時該就醫/急診
4. 說明不確定性：證據不足/族群差異/需醫師評估的條件
5. 自我照護建議必須附「適用條件」與「停止條件」
6. 利益揭露：合作/團購/業配需標示
7. 隱私保護：案例需去識別化

### ❌ 絕對禁止
1. 不做個人化診斷（尤其留言區）
2. 不提供處方用藥劑量
3. 不承諾療效（「一定會好」「保證改善」）
4. 不恐嚇式行銷
5. 不誇大單一研究結論
6. 不污名化特定族群/疾病

## 工作流程

```
1. 確認輸入參數
   ↓
2. 選擇結構模板 → 參考 references/structure-templates.md
   ↓
3. 撰寫初稿（順序：TL;DR → 紅旗 → 主體 → 自我照護）
   ↓
4. 溫度修稿 → 參考 references/tone-guidelines.md
   ↓
5. 安全檢核 → 確認紅旗、免責、無醫囑式用語
   ↓
6. 輸出完整文章
```

## Input Schema

| 欄位 | 說明 | 預設值 |
|------|------|--------|
| topic | 主題（必填） | - |
| audience | 目標讀者 | 一般民眾 |
| platform | 發布平台 | 長文（部落格/FB） |
| emotion | 讀者當下情緒 | 焦慮 |
| action_goal | 希望讀者做的 1 個動作 | 依主題判斷 |
| scope_limit | 不涵蓋的族群/範圍 | 孕婦/幼兒/特殊疾病另行就醫 |
| key_takeaway | 核心訊息一句話 | 依主題生成 |

**最小輸入**：只需 `topic` 即可啟動，其他使用預設值。

## Output Schema

| 欄位 | 說明 |
|------|------|
| title_options | 3 個標題選項（安心型/行動型/破解迷思型） |
| tldr | 3-5 點重點摘要 |
| article | 完整文章（含小標） |
| red_flags | 紅旗清單 |
| disclaimer | 溫柔免責聲明 |
| references | 延伸閱讀來源（選填） |
| comment_templates | 留言回覆模板（選填，用戶要求時提供） |

## 平台適配

### 長文（部落格/FB）
- 字數：1200-2500 字
- 結構：完整 8 段式
- 可包含詳細機轉說明

### 短文（IG/Threads）
- 字數：300-600 字
- 結構：TL;DR + 紅旗 + 一個行動 + 免責
- 用條列式，每點 1-2 句

### 影片稿（Reels/Shorts/YT）
- 時長標註：30秒/1分鐘/3分鐘
- 口語化，加入停頓標記
- 開頭 3 秒要有 hook

## 證據引用原則

- A 級：國家/學會指引、WHO/CDC、系統性回顧（優先使用）
- B 級：大型研究、RCT、隊列研究
- C 級：專家共識、臨床經驗（需標註「經驗性整理」）

文中不塞括號引用，改用「延伸閱讀」區塊放 3-8 個高品質來源。

## 自評 Rubric（完成後檢核）

| 面向 | 0 分 | 1 分 | 2 分 |
|------|------|------|------|
| 溫度 | 無共感 | 有共感但說教 | 共感＋去羞恥＋給選擇 |
| 清晰 | 術語堆砌 | 有翻譯但冗長 | TL;DR 明確、段落短 |
| 專業 | 無來源 | 有來源但無例外 | 有界線＋例外＋紅旗 |
| 安全 | 像醫囑 | 有免責但模糊 | 適用/停止條件明確 |

目標：每項至少 1 分，總分 ≥ 6 分。
