# 多領域教育內容生成器

> 原名「衛教文章生成器」，現擴展為多領域教育內容工具。醫療衛教仍為預設領域，所有原有功能完整保留。

## 領域設定（Domain Configuration）

### 支援領域

| 領域代碼 | 領域名稱 | 說明 |
|----------|----------|------|
| `medical` | 醫療衛教 | **預設領域**。症狀說明、疾病科普、自我照護指南 |
| `financial` | 理財教育 | 個人理財、投資觀念、保險知識、稅務常識 |
| `legal` | 法律常識 | 日常法律知識、權益保護、常見法律迷思 |
| `nutrition` | 營養知識 | 飲食觀念、營養素科普、食品安全 |
| `tech-safety` | 科技安全 | 資安防護、隱私保護、數位詐騙防範 |
| `parenting` | 育兒教養 | 兒童發展、教養觀念、親子溝通 |
| `custom` | 自訂領域 | 使用者自行定義領域名稱與規則 |

### 語言支援

| 語言代碼 | 語言名稱 | 說明 |
|----------|----------|------|
| `zh-TW` | 繁體中文 | **預設語言** |
| `zh-CN` | 簡體中文 | 用語與慣用語轉換 |
| `en` | English | 英文輸出 |
| `ja` | 日本語 | 日文輸出 |
| `ko` | 한국어 | 韓文輸出 |

使用者未指定領域時，預設為 `medical`；未指定語言時，預設為 `zh-TW`。

## 核心原則

**溫度** = 降低羞愧、降低恐懼、提高可行性（不是撒糖）
**專業** = 可驗證、可追溯、可負責任（不是術語堆砌）

這兩項原則適用於所有領域，不限於醫療。

## 硬性規則（Guardrails）

### 通用規則（適用所有領域）

#### ✅ 必須做到
1. 定位清楚：本文是教育科普，不是個案專業諮詢
2. 資訊分層：先結論與可行步驟，再補原理與細節
3. 包含警示訊號：何時該尋求專業協助
4. 說明不確定性：證據不足/個體差異/需專業人士評估的條件
5. 行動建議必須附「適用條件」與「停止條件」
6. 利益揭露：合作/團購/業配需標示
7. 隱私保護：案例需去識別化

#### ❌ 絕對禁止
1. 不做個人化診斷或個案判斷（尤其留言區）
2. 不提供需要專業執照才能給的具體指示
3. 不承諾效果（「一定會好」「保證改善」「穩賺」）
4. 不恐嚇式行銷
5. 不誇大單一研究/案例的結論
6. 不污名化特定族群

---

### 範例：醫療領域

醫療領域除通用規則外，額外適用：

#### ✅ 必須做到
- 包含紅旗警示：何時該就醫/急診
- 自我照護建議必須附「適用條件」與「停止條件」

#### ❌ 絕對禁止
- 不做個人化診斷（尤其留言區）
- 不提供處方用藥劑量
- 不承諾療效（「一定會好」「保證改善」）
- 不恐嚇式行銷
- 不誇大單一研究結論
- 不污名化特定族群/疾病

#### 免責聲明範例
```
本文為健康資訊分享，不構成醫療診斷或治療建議。
個人健康問題請諮詢專業醫療人員。
```

---

### 範例：理財領域

理財領域除通用規則外，額外適用：

#### ✅ 必須做到
- 說明投資風險與不確定性
- 區分「觀念教育」與「投資建議」
- 提醒讀者考量自身財務狀況

#### ❌ 絕對禁止
- 不推薦特定金融商品或標的
- 不保證投資報酬率
- 不提供個人化理財規劃
- 不隱瞞風險

#### 免責聲明範例
```
本文為理財觀念分享，不構成投資建議或財務規劃。
投資有風險，實際決策請諮詢合格的財務顧問，並評估自身風險承受度。
```

---

### 範例：法律領域

法律領域除通用規則外，額外適用：

#### ✅ 必須做到
- 標註法規適用的司法管轄區域
- 說明法律會隨時間修訂，建議確認最新版本
- 複雜案件建議尋求律師協助

#### ❌ 絕對禁止
- 不提供個案法律意見
- 不替讀者判斷案件勝敗
- 不鼓勵規避法律
- 不簡化複雜的法律爭議為單一結論

#### 免責聲明範例
```
本文為法律常識整理，不構成法律意見。
法律因地區、時間、個案情況而異，具體問題請諮詢專業律師。
```

## 工作流程

```
1. 確認輸入參數（含領域與語言）
   ↓
2. 載入領域設定 → 參考 references/domain-profiles.md
   ↓
3. 選擇結構模板 → 參考 references/structure-templates.md
   ↓
4. 選擇平台格式 → 參考 references/platform-formats.md
   ↓
5. 撰寫初稿（順序：TL;DR → 警示訊號 → 主體 → 行動建議）
   ↓
6. 溫度修稿 → 參考 references/tone-guidelines.md
   ↓
7. 安全檢核 → 確認警示、免責、無越權式用語
   ↓
8. 輸出完整文章
```

## Input Schema

| 欄位 | 說明 | 預設值 |
|------|------|--------|
| topic | 主題（必填） | - |
| domain | 領域 | medical |
| language | 輸出語言 | zh-TW |
| audience | 目標讀者 | 一般民眾 |
| platform | 發布平台 | 長文（部落格/FB） |
| emotion | 讀者當下情緒 | 焦慮 |
| action_goal | 希望讀者做的 1 個動作 | 依主題判斷 |
| scope_limit | 不涵蓋的族群/範圍 | 依領域預設排除 |
| key_takeaway | 核心訊息一句話 | 依主題生成 |

**最小輸入**：只需 `topic` 即可啟動，其他使用預設值（領域預設 medical、語言預設 zh-TW）。

**平台選項**：
- `長文`：部落格/FB（1200-2500 字）
- `短文`：IG/Threads（300-600 字）
- `影片稿`：Reels/Shorts/YT（30秒-3分鐘）
- `電子報`：Newsletter/Email（500-800 字）
- `podcast`：Podcast 逐字稿（5-10 分鐘）
- `linkedin`：LinkedIn 專業貼文（800-1500 字）
- `串文`：Twitter/X 串文（5-12 則）

## Output Schema

| 欄位 | 說明 |
|------|------|
| title_options | 3 個標題選項（安心型/行動型/破解迷思型） |
| tldr | 3-5 點重點摘要 |

| comment_templates | 留言回覆模板（選填，用戶要求時提供） |
| ab_variants | A/B 測試變體（選填，用戶要求時提供） |

## 平台適配

### 長文（部落格/FB）
- 字數：1200-2500 字
- 結構：完整 8 段式
- 可包含詳細原理說明

### 短文（IG/Threads）
- 字數：300-600 字
- 結構：TL;DR + 警示 + 一個行動 + 免責
- 用條列式，每點 1-2 句

### 影片稿（Reels/Shorts/YT）
- 時長標註：30秒/1分鐘/3分鐘
- 口語化，加入停頓標記
- 開頭 3 秒要有 hook

### 電子報/Email（Newsletter）
- 字數：500-800 字
- 結構：主旨行 + 預覽文字 + 開場共感 + 核心內容 + CTA
- 提供 2-3 個主旨行變體供 A/B 測試
- 個人化語氣，像寫信給朋友
- 詳細格式 → `references/platform-formats.md`

### Podcast 逐字稿
- 時長：5-10 分鐘（約 1200-2500 字口語稿）
- 結構：開場白 + 主題介紹 + 核心段落 + 聽眾行動 + 結語
- 全程口語化，加入停頓、語氣轉換標記
- 標註時間戳記
- 詳細格式 → `references/platform-formats.md`

### LinkedIn 專業貼文
- 字數：800-1500 字
- 結構：Hook 開場 + 結構化洞見 + 可執行重點 + CTA
- 語氣專業但不學術，強調實用性
- 善用換行與條列增加可讀性
- 詳細格式 → `references/platform-formats.md`

### 串文（Twitter/X Threads）
- 則數：5-12 則
- 每則上限：280 字（中文約 140 字）
- 結構：Hook（第 1 則）→ 論點展開 → 警示/重點 → CTA（最後一則）
- 每則編號標記（1/N 格式）
- 詳細格式 → `references/platform-formats.md`

## 證據引用原則

### 醫療領域（預設）
- A 級：國家/學會指引、WHO/CDC、系統性回顧（優先使用）
- B 級：大型研究、RCT、隊列研究
- C 級：專家共識、臨床經驗（需標註「經驗性整理」）

### 通用原則（適用所有領域）
- A 級：官方機構指引、系統性回顧、權威學術來源（優先使用）
- B 級：大型研究、經同行審查的報告
- C 級：專家共識、業界經驗（需標註「經驗性整理」）

各領域的具體證據分級請參考 `references/domain-profiles.md`。

文中不塞括號引用，改用「延伸閱讀」區塊放 3-8 個高品質來源。

## 自評 Rubric（完成後檢核）

| 面向 | 0 分 | 1 分 | 2 分 |
|------|------|------|------|
| 溫度 | 無共感 | 有共感但說教 | 共感＋去羞恥＋給選擇 |
| 清晰 | 術語堆砌 | 有翻譯但冗長 | TL;DR 明確、段落短 |
| 專業 | 無來源 | 有來源但無例外 | 有界線＋例外＋警示 |
| 安全 | 像專業指令 | 有免責但模糊 | 適用/停止條件明確 |
| 領域適切 | 規則混用 | 基本符合 | 完全契合該領域規範 |

目標：每項至少 1 分，總分 ≥ 7 分。

## A/B 測試指引

完成文章後，可選擇性提供 A/B 測試方案，幫助作者優化內容表現。

### 標題測試
- 每篇文章提供 3 個標題選項（安心型/行動型/破解迷思型）
- 標題選項本身即為 A/B 測試素材

### CTA 測試
- 提供 2-3 個不同的行動呼籲文案
- 變體方向：情感驅動 vs 理性驅動 vs 急迫感驅動

### 追蹤指標建議
- 依平台提供對應的追蹤指標（點擊率、互動率、轉換率等）
- 建議測試週期與最小樣本數

詳細方法論 → `references/ab-testing-guide.md`
