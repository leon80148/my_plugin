# A/B 測試方法論指南

本文件提供教育內容 A/B 測試的完整方法論，幫助內容創作者透過數據驅動的方式優化文章表現。

---

## 核心觀念

### 什麼是內容 A/B 測試

針對同一篇文章的不同元素（標題、CTA、開場白等），產出 2-3 個變體，投放給不同受眾群組，根據數據選出表現最佳的版本。

### 測試三原則

1. **一次只測一個變數**：標題測試時不要同時改 CTA
2. **樣本要夠大**：依平台和內容類型決定最小樣本數
3. **時間要夠長**：至少 24-48 小時，避免時段偏差

---

## 標題測試框架

### 三種標題原型

每篇文章產出 3 個標題選項，各代表不同的心理觸發：

| 原型 | 心理觸發 | 公式 | 範例 |
|------|---------|------|------|
| **安心型** | 降低焦慮、提供確定感 | 「{主題}：{正面結果}的完整指南」 | 「腸躁症：你可以控制症狀的完整指南」 |
| **行動型** | 激發動機、提供方向 | 「{數字} 個 {動作} 來改善你的 {問題}」 | 「5 個日常習慣來改善你的睡眠品質」 |
| **破解迷思型** | 挑戰認知、引發好奇 | 「{常見觀念}？{專家角色}說不一定」 | 「感冒多喝水就會好？醫師說不一定」 |

### 標題測試評分矩陣

| 指標 | 權重 | 測量方式 |
|------|------|----------|
| 開啟率 (Open Rate) | 40% | Email 平台數據 |
| 點擊率 (CTR) | 30% | 連結點擊 / 曝光 |
| 完整閱讀率 | 20% | 滾動深度 / 閱讀時間 |
| 分享率 | 10% | 社群分享次數 |

### 判定規則

- **顯著差異**：變體間差異 > 5% 且 p-value < 0.05
- **無顯著差異**：選擇品牌調性最一致的版本
- **樣本不足**：延長測試時間或降低信心水準至 90%

## CTA 測試框架

### CTA 三種風格

| 風格 | 驅動力 | 範例 |
|------|--------|------|
| **情感驅動** | 共感、歸屬感 | 「為了家人的健康，今天就開始改變」 |
| **理性驅動** | 數據、邏輯 | 「根據研究，定期篩檢可降低 40% 風險」 |
| **急迫感驅動** | 時間壓力 | 「免費篩檢只到年底，立即預約」 |

### CTA 位置測試

| 位置 | 適用情境 | 預期效果 |
|------|----------|----------|
| 文章中段 | 讀者已理解問題但尚未疲勞 | 較高轉換率 |
| 文章末尾 | 讀者已充分理解且被說服 | 較高品質轉換 |
| 懸浮按鈕 | 行動裝置長文 | 較高曝光率 |
| 內文連結 | 自然融入閱讀流 | 較低侵入感 |

### 基準值使用方式

1. 先確認你的平台和內容類型
2. 查找對應的基準值
3. 如果目前低於基準 → 優先優化該指標
4. 如果已達良好 → 考慮測試其他變數

## 測試流程（Step by Step）

### Phase 1：規劃（發布前）

1. **選擇測試變數**：標題 / CTA / 開場白 / 圖片
2. **產出變體**：2-3 個版本
3. **設定目標指標**：選擇 1 個主要指標 + 1-2 個輔助指標
4. **計算最小樣本數**：
   - Email：至少 1,000 收件人/組
   - 社群貼文：至少 5,000 曝光/組
   - 部落格：至少 500 訪客/組

### Phase 2：執行（發布中）

1. **隨機分組**：確保受眾隨機分配
2. **同時投放**：所有變體在同一時間發布
3. **不提前判定**：至少等待 24-48 小時
4. **監控異常**：檢查是否有技術問題導致數據偏差

### Phase 3：分析（發布後）

1. **數據收集**：匯出各變體的指標數據
2. **統計檢定**：確認差異是否顯著
3. **結果解讀**：
   - 勝出變體 → 採用並記錄原因
   - 無顯著差異 → 選品牌調性最符的版本
   - 意外結果 → 深入分析，可能發現新洞見
4. **知識累積**：將結果記錄到「A/B 測試知識庫」

## 常見測試組合建議

### 新手起步（先測這些）

| 優先級 | 測試變數 | 理由 |
|--------|---------|------|
| 1 | 標題 | 影響最大，決定是否被點開 |
| 2 | CTA 文案 | 決定轉換率 |
| 3 | 開場白 | 決定是否繼續閱讀 |

### 進階測試

| 測試變數 | 適用平台 | 注意事項 |
|---------|---------|---------|
| 發佈時間 | 全平台 | 同內容不同時段 |
| 文章長度 | Blog、LinkedIn | 長短版對照 |
| 圖片 vs 無圖片 | 社群 | 視覺吸引力測試 |
| emoji 使用 | Newsletter、社群 | 有無 emoji 對比 |
| 免責聲明位置 | 全平台 | 前置 vs 後置 |

## 避免常見錯誤

| 錯誤 | 問題 | 修正 |
|------|------|------|
| 同時改多個變數 | 無法歸因哪個變數造成差異 | 一次只改一個 |
| 樣本太小就下結論 | 結果可能是隨機波動 | 等達最小樣本數 |
| 只看一個指標 | 可能提高點擊但降低信任 | 主要 + 輔助指標 |
| 忽略季節/時事 | 外部因素干擾結果 | 記錄測試期間的外部事件 |
| 不記錄結果 | 無法累積學習 | 每次測試都填寫報告模板 |
| 測試期間修改內容 | 數據汙染 | 一旦上線不要改 |
