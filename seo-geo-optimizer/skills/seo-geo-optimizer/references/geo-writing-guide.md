# GEO 寫作指南與分析評分細則

本文件為 seo-geo-optimizer skill 的寫作規範與分析評分細則參考。

## 一、GEO 寫作完整規範

### 1.1 倒金字塔寫法詳解

傳統寫法 vs GEO 寫法的核心差異在於資訊排列順序。GEO 要求每個段落在前 50-75 字內提供直接、可被 AI 獨立擷取的解答。

**反面案例（傳統寫法）**：
```
隨著雲端運算技術的快速發展，越來越多企業開始關注如何
有效管理雲端支出。根據多項調查顯示，雲端成本已經成為
IT 主管最關心的議題之一。Flexera 在 2024 年的報告中
指出，73% 的企業表示雲端支出超出預算。
```

**正面案例（GEO 寫法）**：
```
73% 的企業雲端支出超出預算（Flexera, 2024 State of
the Cloud Report）。最常見的三大浪費來源為閒置資源
（佔 32%）、過度配置（佔 26%）和缺乏自動排程（佔 18%）。
```

**差異分析**：
- GEO 版本首句即為可引用的事實主張
- 數據帶完整來源歸屬
- 無冗餘修飾語（「隨著...發展」「越來越多」）
- AI 可從此段獨立提取完整答案，無需上下文

### 1.2 統計數據嵌入規範

每 200-300 字必須至少嵌入 1 個帶來源的數據點。數據品質層級：

| 層級 | 來源類型 | 權威度 | 範例 |
|------|----------|--------|------|
| S 級 | 同行審查學術論文 | 最高 | 「根據 Nature (2024) 的研究...」 |
| A 級 | 政府/國際組織報告 | 高 | 「WHO 2024 年報告指出...」 |
| B 級 | 權威產業研究機構 | 中高 | 「Gartner 預測...」「IDC 資料顯示...」 |
| C 級 | 知名媒體調查 | 中 | 「根據 Bloomberg 報導...」 |
| D 級 | 企業自有數據 | 中低 | 「我們分析了 10,000 筆客戶資料...」 |

**數據嵌入格式**：
```
[數值] + [單位/百分比] + [時間範圍] + [來源機構] + [報告名稱/年份]
```

範例：「全球 AI 市場規模預計從 2024 年的 1,840 億美元成長至 2030 年的 8,267 億美元（Statista, AI Market Revenue Forecast, 2024）」

### 1.3 專家語錄嵌入規範

每篇核心內容至少 1-2 則具名專家語錄。語錄必須包含完整歸屬特徵：

**完整格式**：
```
據 [機構名稱] [姓名]（[頭銜]）指出：「[直接引述內容]」
```

**歸屬特徵檢查清單**：
- 專家全名（非匿名）
- 現任職稱/頭銜
- 所屬機構/組織
- 相關專業背景（選填但建議）

**反面案例**：
- 「有專家指出...」（匿名，無法驗證）
- 「業界人士表示...」（模糊歸屬）
- 「據了解...」（無來源）

**正面案例**：
- 「據 Google 搜尋品質團隊負責人 Danny Sullivan 表示：『我們將持續強化 AI Overviews 的引用來源透明度。』」

### 1.4 權威引用連結規範

仿照學術論文模式，為所有關鍵事實主張附上信任鏈結：

**引用格式建議**：
```
行內引用：... 達成 40% 的效率提升 [來源: McKinsey, 2024]。
段末引用：（資料來源：WHO Global Health Report 2024, p.32）
集中引用：文末設置「參考來源」區塊，編號對應正文
```

**引用來源權威層級排序**：
1. 學術論文（PubMed、Google Scholar、IEEE）
2. 政府/國際組織官方報告
3. 權威媒體（Reuters、Bloomberg、NYT）
4. 產業研究機構（Gartner、IDC、Forrester）
5. 知名企業公開數據
6. 專業部落格/社群媒體（最低層級）

### 1.5 語言雜訊清除清單

以下用語在 GEO 寫作中必須避免或替換：

| 禁止用語 | 問題 | 替代方案 |
|----------|------|----------|
| 眾所周知 | 空泛，無資訊增益 | 直接陳述事實 + 數據 |
| 隨著科技發展 | 陳腔濫調 | 指出具體技術和時間點 |
| 越來越多人 | 模糊不精確 | 引用具體調查數據 |
| 值得注意的是 | 冗餘過渡語 | 直接陳述 |
| 不可否認地 | 主觀修飾 | 刪除，直述事實 |
| 毫無疑問 | 武斷判斷 | 提供支持證據 |
| 在這個時代 | 空洞開場 | 直接切入主題 |
| 據了解/據悉 | 匿名來源 | 明確指出消息來源 |

## 二、六大維度評分細則

### 2.1 GEO 三大驅動因子（0-10 分）

| 分數 | 標準 |
|------|------|
| 9-10 | 三項因子齊全；統計數據每 200 字 >= 1 個，來源 B 級以上；專家語錄 >= 2 則含完整歸屬；權威引用 >= 5 處含連結 |
| 7-8 | 三項因子齊全但密度略低；數據每 400 字 1 個或部分缺來源 |
| 5-6 | 具備 2 項因子；第 3 項因子明顯不足 |
| 3-4 | 僅具備 1 項因子；整體數據/引用稀疏 |
| 1-2 | 因子模糊，無明確數據、語錄或來源 |
| 0 | 完全缺乏三大因子 |

### 2.2 結構與機器可讀性（0-10 分）

| 分數 | 標準 |
|------|------|
| 9-10 | 全文倒金字塔結構；H2/H3 層次清晰；每段自包含可獨立理解；有項目符號輔助 |
| 7-8 | 大部分段落倒金字塔；標題層次合理但偶有跳級 |
| 5-6 | 部分段落倒金字塔；標題結構存在但不一致 |
| 3-4 | 傳統順敘結構為主；關鍵資訊埋在段落中間 |
| 1-2 | 無明確結構；大段純文字無分段 |
| 0 | 完全無結構化元素 |

### 2.3 E-E-A-T 信號強度（0-10 分）

| 分數 | 標準 |
|------|------|
| 9-10 | 四項信號齊全：有第一手經驗、作者可追溯、第三方背書、信任信號完整 |
| 7-8 | 具備 3 項信號且品質良好 |
| 5-6 | 具備 2 項信號 |
| 3-4 | 僅具備 1 項信號 |
| 1-2 | 信號模糊不清 |
| 0 | 完全無 E-E-A-T 信號 |

### 2.4 傳統 SEO 基礎健康度（0-10 分）

| 分數 | 標準 |
|------|------|
| 9-10 | Title/Meta 完善；URL 乾淨；圖片 Alt Text 完整；內部連結合理；CWV 達標 |
| 7-8 | 大部分 SEO 元素到位，1-2 項可優化 |
| 5-6 | 基礎 SEO 元素存在但品質參差 |
| 3-4 | 多項 SEO 元素缺失或品質差 |
| 1-2 | SEO 基礎幾乎未建置 |
| 0 | 完全無 SEO 優化痕跡 |

### 2.5 對話意圖覆蓋度（0-10 分）

| 分數 | 標準 |
|------|------|
| 9-10 | 覆蓋四維度查詢；預測 5+ 個追問並提供解答；支援 10-23 字長尾查詢 |
| 7-8 | 覆蓋 3 個維度；有追問預測但不完整 |
| 5-6 | 覆蓋 2 個維度 |
| 3-4 | 僅覆蓋 1 個維度 |
| 1-2 | 內容僅回答單一問題 |
| 0 | 無法匹配任何對話查詢意圖 |

### 2.6 多模態就緒度（0-10 分）

| 分數 | 標準 |
|------|------|
| 9-10 | 圖像 Alt Text 完整 + EXIF 中介資料；影片有字幕 + 章節 + Schema；字體/對比符合標準 |
| 7-8 | 大部分多模態元素到位 |
| 5-6 | 圖像有 Alt Text 但品質參差；影片缺字幕 |
| 3-4 | 多模態元素稀少 |
| 1-2 | 幾乎無多模態最佳化 |
| 0 | 純文字內容無任何多模態元素 / 不適用 |

## 三、YMYL 領域 E-E-A-T 強化檢查清單

YMYL（Your Money or Your Life）領域的內容需通過更嚴格的審查：

### 醫療健康類

- [ ] 所有醫療主張引用同行審查文獻或官方指引
- [ ] 作者醫療資格聲明（學歷、證照號碼、執業年資）
- [ ] Person Schema 標記包含 hasCredential
- [ ] 免責聲明：「本文僅供參考，不構成醫療建議」
- [ ] 所有引用時間戳記在 12 個月內
- [ ] 藥物/治療方案附上禁忌症與副作用說明

### 金融投資類

- [ ] 投資建議附上風險警告
- [ ] 財務數據引用官方財報或權威金融資料庫
- [ ] 作者金融證照聲明（CFA、CFP 等）
- [ ] 免責聲明：「本文不構成投資建議」
- [ ] 市場數據標註即時性（「截至 2024 年 Q3」）

### 法律類

- [ ] 法律意見標註適用管轄區域
- [ ] 引用具體法條/判例
- [ ] 作者律師資格聲明
- [ ] 免責聲明：「本文不構成法律意見」
