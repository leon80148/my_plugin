# 策略規劃框架與 KPI 體系

本文件為 seo-geo-optimizer skill 模式 D（策略規劃）的完整展開參考。

## 一、現狀診斷方法論

### 1.1 傳統 SEO 健康度評估

執行以下技術稽核項目：

| 稽核項目 | 工具 | 合格標準 |
|----------|------|----------|
| Core Web Vitals | PageSpeed Insights | LCP < 2.5s, FID < 100ms, CLS < 0.1 |
| 行動裝置友善度 | Google Mobile-Friendly Test | 通過所有測試 |
| 索引狀態 | Google Search Console | 索引頁面數 / 提交頁面數 > 90% |
| 結構化資料 | Rich Results Test | 零錯誤，警告 < 5 |
| HTTPS 部署 | SSL Labs | A 級以上 |
| 內部連結架構 | Screaming Frog | 孤立頁面 < 5%，爬取深度 < 4 |
| 重複內容 | Siteliner | 重複比例 < 15% |

### 1.2 AI 能見度基準線測量

使用「提示詞矩陣法」手動測量品牌在主要 AI 平台的能見度：

**步驟**：
1. 建立 20-30 個目標提示詞（見 D2 提示詞矩陣）
2. 在 ChatGPT、Perplexity、Gemini、Claude、Google AI Overviews 逐一測試
3. 記錄以下指標：
   - 品牌是否被提及（是/否）
   - 提及位置（首選推薦 / 列表中 / 備註）
   - 情感傾向（正面 / 中性 / 負面）
   - 資訊準確度（正確 / 部分正確 / 錯誤）
   - 引用來源是否指向品牌官網

**基準線記錄表格式**：

| 提示詞 | ChatGPT | Perplexity | Gemini | Claude | AI Overviews |
|--------|---------|------------|--------|--------|--------------|
| [提示詞1] | 提及/位置/情感 | ... | ... | ... | ... |
| [提示詞2] | ... | ... | ... | ... | ... |

### 1.3 競爭者 AI 引用分析

對前 3-5 名競爭者執行相同的提示詞矩陣測試，建立對比矩陣：

| 品牌 | 引用次數 | 首選推薦次數 | 正面情感比例 | 平均引用位置 |
|------|----------|--------------|-------------|-------------|
| 我方品牌 | _ | _ | _% | _ |
| 競爭者 A | _ | _ | _% | _ |
| 競爭者 B | _ | _ | _% | _ |

## 二、內容工程改造計畫

### 2.1 頁面 GEO 改造優先序

使用「商業價值 x 改造難度」矩陣決定優先序：

| 優先級 | 頁面類型 | 改造重點 |
|--------|----------|----------|
| P0（立即） | 核心營收頁面（產品頁、定價頁） | 倒金字塔重構 + Schema + Markdown 鏡像 |
| P1（2週內） | 高流量內容頁（部落格 Top 20） | 統計數據嵌入 + 專家語錄 + FAQ Schema |
| P2（1月內） | 品牌信任頁（關於我們、團隊頁） | E-E-A-T 強化 + Person Schema |
| P3（持續） | 長尾內容（所有部落格文章） | 對話意圖覆蓋 + 追問預測 |

### 2.2 系統性嵌入計畫

為現有內容建立「GEO 三大因子注入」的作業流程：

**統計數據注入流程**：
1. 盤點現有內容中的模糊描述（「很多」「大幅成長」）
2. 查找對應的精確數據和來源
3. 替換為帶來源的數據陳述
4. 每篇至少達成每 300 字 1 個數據點的密度

**專家語錄注入流程**：
1. 建立領域專家資料庫（姓名、頭銜、機構、專長）
2. 聯繫專家取得引用授權或使用公開演講/發表內容
3. 每篇核心內容嵌入 1-2 則語錄
4. 確保歸屬特徵完整（姓名 + 頭銜 + 機構）

**權威引用注入流程**：
1. 為每篇文章的關鍵主張列出支持來源
2. 優先使用 S-B 級來源（學術論文、政府報告、產業研究）
3. 附上連結或完整引用資訊
4. 定期檢查連結是否失效（建議季度稽核）

## 三、技術架構升級路線

### 3.1 實施時程建議

| 階段 | 時程 | 工作項目 |
|------|------|----------|
| Phase 1 | 第 1-2 週 | 部署 llms.txt + Organization Schema |
| Phase 2 | 第 3-4 週 | 為核心頁面建立 Markdown 鏡像 + FAQPage Schema |
| Phase 3 | 第 5-6 週 | 為所有作者建立 Person Schema + Article Schema |
| Phase 4 | 第 7-8 週 | 渲染障礙排除（減少 JS 依賴、確保純文本可提取） |
| 持續 | 每月 | 內容更新時同步更新 Schema 的 dateModified |

### 3.2 渲染障礙排除清單

AI 爬蟲解析能力有限，以下為常見障礙及解決方案：

| 障礙 | 問題 | 解決方案 |
|------|------|----------|
| 客戶端渲染 (CSR) | AI 爬蟲可能無法執行 JavaScript | 改用 SSR/SSG 或預渲染 |
| 無限捲動 | 內容無法被完整爬取 | 提供分頁替代方案 |
| 登入牆 | AI 爬蟲無法存取 | 核心內容保持公開 |
| iframe 嵌入 | 嵌入內容無法被解析 | 在主頁面內聯內容 |
| 圖片內文字 | OCR 準確度不穩定 | 提供文字替代（Alt Text / 隱藏文字） |
| 動態表格 | JS 產生的表格難以解析 | 提供 Markdown 鏡像 |

## 四、知識圖譜建構策略 (KGO)

品牌提及與 AI 可見度相關性（0.664）遠高於反向連結（0.218），因此知識圖譜建構是 GEO 最有效的長期策略。

### 4.1 NAP 一致性稽核

NAP = Name（名稱）、Address（地址）、Phone（電話）

- 確保品牌名稱在所有線上平台完全一致（含標點符號）
- Google Business Profile、Apple Maps、Bing Places 資訊同步
- 社群媒體帳號名稱統一
- 目錄網站（黃頁、產業目錄）資訊更新

### 4.2 Wikidata 條目建立

Wikidata 是多數 AI 引擎的知識圖譜基礎來源：

1. 確認品牌是否已有 Wikidata 條目
2. 若無，準備可靠來源（新聞報導、企業登記）作為佐證
3. 建立條目，填入：instance of、official website、inception date、industry、headquarters location
4. 連結至 Wikipedia 條目（若存在）
5. 定期更新條目資訊

### 4.3 數位公關策略

提升品牌在第三方權威內容中的「共現信號」：

| 策略 | 目標 | 方法 |
|------|------|------|
| 產業最佳清單 | 被收錄進「Top 10 [產業] 工具」類文章 | 主動聯繫作者、提供試用帳號 |
| 對比評測 | 出現在「[品牌A] vs [品牌B]」類型分析中 | 提供完整產品資訊與差異化優勢 |
| 產業生態圖 | 被標示在市場地圖/象限圖中 | 聯繫分析師、提交產品資訊 |
| 客座發表 | 在權威媒體/部落格發表署名文章 | 產出高品質原創研究/洞見 |
| 案例研究 | 客戶在公開場合引用品牌 | 協助客戶撰寫成功案例 |

### 4.4 品牌共現信號培養

品牌共現（Brand Co-occurrence）指品牌名稱與特定產業/問題關鍵字在第三方內容中反覆同時出現。

**實施方法**：
- 確保品牌名稱與目標產業關鍵字在同一文章/段落中共現
- 在客座發表、新聞稿、合作內容中刻意建立共現模式
- 監控共現頻率（使用 Google Alerts 或 Mention）

## 五、多模態搜尋布局

### 5.1 圖像 AI 可讀性規範

| 規範項目 | 標準 | 說明 |
|----------|------|------|
| Alt Text | 描述性、含關鍵字 | 描述圖片內容而非裝飾用途 |
| EXIF 中介資料 | Title, Description, Keywords | 嵌入圖片檔案本身 |
| IPTC 中介資料 | Caption, Credit, Source | 專業圖片管理標準 |
| 字體高度 | > 30px | 確保 OCR 可辨識 |
| 灰階對比度 | > 40 | 確保文字/背景區分度 |
| 情感對齊 | 圖文一致 | 圖片情感與文字內容相符 |

### 5.2 影片 AI 解析規範

| 規範項目 | 格式 | 說明 |
|----------|------|------|
| 字幕檔 | WebVTT (.vtt) | 必須提供完整轉錄，非自動字幕 |
| 時間軸章節 | YouTube Chapters / Schema | 每章節標題描述該段內容 |
| VideoObject Schema | JSON-LD | 含 name, description, duration, thumbnailUrl |
| 轉錄文字頁面 | 獨立 URL | 影片內容的完整文字版本 |

## 六、內容衰退偵測與更新策略

內容衰退（Content Decay）指已發佈內容隨時間流逝而逐漸失去搜尋排名和 AI 引用率的現象。在 GEO 時代，內容衰退的影響比傳統 SEO 更嚴重 — AI 引擎偏好最新資訊，過時內容會被直接跳過。

### 6.1 衰退信號偵測

| 衰退信號 | 偵測方法 | 危險閾值 |
|----------|----------|----------|
| 流量下降 | Google Search Console 月對月比較 | 連續 2 個月下降 > 20% |
| AI 引用消失 | 提示詞矩陣月度測試 | 原本被引用的提示詞連續 2 月未引用 |
| 資料過時 | 檢查引用來源的發佈日期 | 引用資料超過 18 個月 |
| 競爭者超越 | 相同提示詞測試競爭者出現頻率 | 競爭者新內容發佈後 1 個月內 |
| Schema 失效 | Google Rich Results Test | 出現錯誤或警告增加 |
| 外部連結失效 | 定期連結檢查（季度） | 任何 404 連結 |

### 6.2 更新優先序矩陣

| 優先級 | 條件 | 更新動作 | 時限 |
|--------|------|----------|------|
| P0 緊急 | YMYL 內容 + 資料過時 | 全面改寫 + 更新所有引用 | 1 週內 |
| P1 高 | 核心營收頁面 + 流量下降 > 30% | 重構 GEO 三大因子 + 更新數據 | 2 週內 |
| P2 中 | AI 引用消失 + 競爭者超越 | 補充新數據/專家語錄 + 追問預測 | 1 個月內 |
| P3 低 | 一般內容 + 輕微流量下降 | 更新時間戳 + 刷新引用來源 | 下次排程 |

### 6.3 系統性更新流程

**季度內容稽核流程**：

1. **匯出清單** — 從 CMS 匯出所有已發佈文章，標註發佈日期和最後修改日期
2. **流量比對** — 與 Google Search Console 數據交叉比對，標記流量下降的頁面
3. **AI 引用檢查** — 針對高價值頁面執行提示詞矩陣測試，確認 AI 引用狀態
4. **來源有效性** — 批次檢查所有外部引用連結是否仍有效
5. **競爭者掃描** — 檢查前 3 名競爭者是否發佈了相同主題的新內容
6. **排定更新** — 依優先序矩陣排入更新排程

**更新後必做**：
- 更新 `dateModified`（Schema + 頁面可見處）
- 重新提交 Sitemap
- 在 llms.txt / llms-full.txt 中反映變更
- 於社群分享更新版本以觸發重新爬取

### 6.4 常青內容策略

將容易衰退的「時效性內容」轉化為持續有效的「常青內容」：

| 時效性寫法 | 常青寫法 |
|-----------|---------|
| 「2024 年最新的 10 大工具」 | 「[動態年份] 最佳工具推薦（持續更新）」 |
| 「根據最新研究顯示...」 | 「根據 [機構] [年份] 研究顯示...」（易於定期替換） |
| 「目前市場價格約 NT$500」 | 「市場價格約 NT$500（截至 [年月]，定期更新）」 |

**關鍵原則**：設計內容結構時，將容易過時的元素（數字、年份、排名）隔離在易替換的位置，而非混入段落敘述中。

## 七、KPI 與追蹤體系

### 7.1 核心 KPI 定義

| KPI | 定義 | 目標 | 測量頻率 |
|-----|------|------|----------|
| AIGVR（AI 生成可見度比率） | 品牌在 AI 回覆中被提及的比率 | YMYL 30-40%；一般 20-30% | 每月 |
| 引用位置 | 品牌在 AI 回覆中的排序位置 | Top 3 | 每月 |
| SOV（聲量份額） | 品牌引用次數 / 品類總引用次數 | 前 3 名 | 每月 |
| 情感傾向 | AI 引用中正面/中性/負面的比例 | 正面 > 70% | 每月 |
| 資訊準確度 | AI 回覆中品牌資訊正確的比例 | > 95% | 每月 |
| 對話式轉換率 | 從 AI 引用連結進站的轉換率 | 基準 x 4.4 倍 | 每週 |

### 7.2 追蹤工具選型矩陣

| 工具 | 適用規模 | 核心功能 | 價格帶 | 合規認證 |
|------|----------|----------|--------|----------|
| Atomic AGI | 大型企業 | AI 優先歸因、多觸點分析 | $$$$ | GDPR |
| Profound AI | 大型企業 | 對話探索器、深度語意分析 | $$$$ | SOC 2 Type II |
| Peec AI | 中小品牌 | 多引擎追蹤、4 小時更新 | $$ | — |
| Ahrefs Brand Radar | SEO 團隊 | 整合 Ahrefs 數據、品牌監控 | $$$ | — |
| SE Visible | 行銷主管 | 直觀圖表、品牌聲譽預警 | $$ | — |

### 7.3 手動追蹤替代方案

若預算有限，可使用以下免費/低成本替代：

1. **提示詞矩陣表格**：用 Google Sheets 手動記錄每月 AI 引用變化
2. **Google Alerts**：監控品牌名稱在網路上的提及
3. **Google Search Console**：追蹤 AI Overviews 帶來的流量（篩選 referrer）
4. **UTM 標記**：在 llms.txt 和 Schema 中的 URL 加入 UTM 參數追蹤 AI 來源流量

### 7.4 報告週期與格式

| 報告 | 週期 | 內容 |
|------|------|------|
| 週報 | 每週 | 對話式轉換率、異常事件 |
| 月報 | 每月 | 全部 6 項 KPI + 趨勢分析 + 競爭者對比 |
| 季報 | 每季 | 策略檢討 + 路線圖調整 + ROI 計算 |

**月報建議結構**：
1. 執行摘要（1 頁）
2. KPI 儀表板（AIGVR、SOV、情感、準確度）
3. 提示詞矩陣結果變化
4. 競爭者動態
5. 已完成改造項目與成效
6. 下月行動計畫
